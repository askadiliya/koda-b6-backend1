# Backend

Backend adalah bagian aplikasi yang berjalan di server, bertanggung jawab untuk memproser logika bisnis, mengelola data base dll
1. menangani permintaan reques dari frondend
2. memproses data CRUD
3. Berinteraksi dengan database
4. mengembalikan respon ke ke frondend

# REST
arsitektur untuk membuat web services

RESTful API Adalah API yang mengikuti prinsip REST

# Karakteristik REST
1. Stateless : setiap reques harus memuat atau mengandung semua informasi yang diperlukan
2. Client-server : pemisah antara frondend dan backend
3. Chacheable : respon dapat disimpan dari chace untuk meningkatkan performa
4. Uniform interface : konsisten dalam struktur request dan respon
5. layered system: arsitektur berlapis (load balancer)

# pemanfaatan http methot
1. GET : Mengambil data (Tidak memiliki Body)
2. POST : mengirim/membuat data baru
3. PUT : Memperbarui seluruh data 
4. PATCH: memperbarui sebagian data
5. DELETE : Menghapus data

# HTTP (HyperText Transfer Protocol)
1. protokol untuk komunikasi antar client dan server
2. menggunakan port 80
3. data terkirim dalam bentuk plain text (tidak terenkripsi)
